1. Executive Summary

India's options and futures market (NSE F&O) is one of the largest derivatives markets in the world by volume, yet the tools available to retail traders range from inadequate to actively misleading. Platforms like Sensibull provide Greeks and strategy payoff charts, but they stop short of explaining market forces in plain language — leaving traders to guess why a strategy that looked perfect on paper went wrong in live conditions.

OPTIQ changes that. It is a browser-based trading intelligence platform purpose-built for Indian equity derivatives (NSE/BSE Stock Options, Nifty/BankNifty options, and F&O segments). The platform combines:

  • A Rust-powered signal and backtesting engine for speed and accuracy
  • An AI layer that translates strategy outcomes into plain-language explanations — not jargon
  • Real-time, uninterrupted live market data sourced from NSE/BSE via Indian data providers
  • A full strategy builder that covers payoffs, Greeks, IV analysis, and scenario simulation
  • A free tier designed to acquire users with zero upfront infrastructure cost, scaling to paid plans as the user base grows

The product is designed for a non-technical founder starting lean, validating with free-tier infrastructure, and scaling to a sustainable SaaS business targeting Indian retail F&O traders — a market of over 45 million active derivatives traders and growing.

2. Problem Statement

Indian retail F&O traders face three compounding problems that no existing platform solves together:

2.1 Data Interruption & Reliability

Most retail-grade platforms (including free broker APIs) suffer from feed drops, WebSocket disconnections, and rate limits that cause strategies to miss signals or execute at wrong prices. For F&O traders where a 5-second delay during expiry can mean the difference between profit and ruin, this is a critical failure point.

2.2 Strategy Understanding Gap

When a strategy fails, platforms show P&L graphs but never explain what actually happened — was it IV crush post-earnings? A sudden shift in Put-Call Ratio? Institutional block deals that moved spot? Retail traders are left reading forums and YouTube instead of learning from their own data. This is the deepest and most underserved problem in the space.

2.3 Speed Limitations in Backtesting

Python-based backtesting (the industry default) is slow. Running a 12-month backtest on 0DTE Nifty options with minute-level data can take 15-40 minutes. This discourages iteration. A Rust engine brings this down to under 30 seconds, enabling traders to test dozens of hypotheses in a single session.

5. Core Feature Modules

MODULE 1 — Live Data Engine

5.1 Uninterrupted Live Market Data

This is the hardest engineering challenge and the most critical reliability requirement. The platform must never drop data during market hours (9:15 AM – 3:30 PM IST), especially around high-volatility events like RBI decisions, Budget announcements, and monthly expiries.

Data Source Strategy

Primary: Upstox or Zerodha WebSocket API (free tier available). Fallback: NSE official data feed or Dhan API. Secondary fallback: Cached 1-minute OHLCV from TrueData (₹800/month). The Rust engine manages reconnection logic and fills gaps automatically.

  • Real-time option chain streaming: All strikes across current and next expiry
  • Live Open Interest (OI) data with change tracking: Critical for detecting institutional positioning
  • Implied Volatility (IV) per strike in real-time with IV percentile vs. 1-year history
  • Put-Call Ratio (PCR) streaming at index and stock level
  • India VIX live feed with rate of change alerts
  • FII/DII provisional data integration (updated every 30 min via NSE website scrape)
  • Auto-reconnect: Rust WebSocket client reconnects within 200ms of any drop
  • Data gap filler: If a candle is missed, it reconstructs from tick data or flags for the user

MODULE 2 — Rust Signal & Backtesting Engine

5.2 The Rust Performance Core

The signal engine is written in Rust — a language that is 10 to 100 times faster than Python for numerical computation. This engine runs entirely on the server, invisible to the user, but responsible for all the heavy lifting: backtesting, signal computation, Greeks calculation, and scenario simulation.

Why Rust?

A Python backtest on 12 months of Nifty minute data takes 15–40 minutes. The same computation in Rust takes under 30 seconds. This means a trader can run 30 backtests in the time it used to take to run one — enabling real learning and iteration.

Backtesting Capabilities

  • Historical data: NSE EOD data (free) + intraday 1-min data from data providers
  • Multi-leg option strategy backtesting: Iron Condor, Straddle, Strangle, Butterfly, Calendar, Ratio spreads
  • Greeks simulation across time: See how Delta, Theta, Vega evolved minute by minute
  • IV surface reconstruction: Recreate the IV smile on any historical date
  • Slippage and brokerage modelling: Built-in Zerodha/Upstox cost structure
  • Expiry-level analysis: Track pinning behavior, gamma risk near expiry, and IV crush post-event

Signal Engine

  • Rule-based signals: User-defined conditions (PCR > 1.3, IV < 20th percentile, price above 20 EMA, etc.)
  • AI-enhanced signals: ML model trained on NSE historical data to score signal confidence from 0–100
  • Signal attribution: Every signal is tagged with contributing factors so the AI can explain it
  • Paper trading mode: All signals can be paper-traded before going live

MODULE 3 — AI Intelligence Layer

5.3 The AI Co-Pilot — The Core Differentiator

This is what no competitor does. Every strategy, every backtest, every live signal comes with a plain-English explanation. Not 'IV Rank is 23'. But: 'Implied Volatility is unusually low right now — cheaper than 77% of the last year. This means option premiums are cheap, making it a good time to buy options rather than sell them. If a big move happens, your bought options gain value faster than they decay.'

AI Philosophy

Every output from OPTIQ answers three questions: What happened? Why did it happen (in terms of market forces)? What does it mean for my position? No jargon. No assumption of prior knowledge. Plain, specific, contextual.

Post-Trade / Post-Backtest Explanation Engine

  • Strategy post-mortem: After a backtest, AI writes a 300-500 word summary explaining why the strategy worked or failed
  • Failure attribution: Was it IV crush? Wrong expiry selection? Adverse delta move? Gap-up open? The AI identifies and ranks contributing factors
  • Market forces narrative: Plain-English explanation of what institutional players, FIIs, or event-driven forces were doing during the period
  • What could have been done differently: The AI suggests 2–3 modifications to the strategy that would have improved outcome, backed by data

Live Market Intelligence

  • Daily pre-market brief (6:30 AM): AI-written 400-word summary of overnight global cues, India VIX outlook, key levels for Nifty/BankNifty, and strategy suggestions for the day
  • Real-time market force alerts: 'FIIs sold 8,200 crore in index futures today — PCR has dropped from 1.4 to 0.9 in 2 hours. Bullish bets are unwinding. Monitor your short puts.'
  • Expiry countdown intelligence: Week-of-expiry daily brief on pinning candidates, gamma walls, and expected move range
  • Earnings event playbook: Before any stock result, AI generates a plain-English guide on how to trade it using options (IV expansion, straddle break-even, etc.)

MODULE 4 — Strategy Builder

5.4 Visual Strategy Builder

A no-code interface for building multi-leg F&O strategies. The user selects instrument, expiry, strike, and action. The platform instantly shows payoff, Greeks, max profit, max loss, break-evens, and probability of profit.

  • Pre-built strategy templates: 25+ common strategies (Covered Call, Protective Put, Iron Condor, Jade Lizard, Broken Wing Butterfly, Calendar Spread, etc.)
  • AI explains each leg: Hover over any leg and the AI explains its role — 'This short call caps your upside but reduces cost. It profits if the stock stays below ₹2,400 till expiry.'
  • Live Greeks panel: Real-time Delta, Theta, Vega, Gamma, Rho for the entire position
  • Scenario simulator: 'What if Nifty falls 300 points in 2 days and IV spikes 5%?' — instant visual answer
  • Adjustment suggestions: AI suggests adjustments when a live trade goes against you — 'Your short strangle has moved into loss. Here are 3 ways to adjust: roll the tested leg, add a hedge, or exit the tested side.'

MODULE 5 — Education & Understanding Layer

5.5 Market Forces Education (Built Into Every Feature)

Rather than a separate 'learn' section, OPTIQ embeds education into every user action. This is the feature that converts casual users into loyal, paying subscribers — because they feel they are actually getting smarter using the platform.

  • Concept tooltips: Every Greek, every indicator, every term has a 2-sentence plain-English tooltip. No link to an external article. Right there.
  • Trade journal with AI review: Users log trades, and the AI retrospectively explains what the market was doing during that period
  • Weekly market debrief: Every weekend, AI generates a 600-word analysis of what drove the market that week — in the style of a smart friend explaining it over chai
  • 'Why did this happen?' button: On any chart, any data point — user can click and ask. AI explains in context.

6.2 Data Pipeline Architecture

The data pipeline is the backbone of the platform and must be fault-tolerant by design. Here is how it works from market open to close:

  • Market open (9:15 AM): Rust WebSocket client connects to Upstox/Zerodha feed and begins streaming tick data for all subscribed instruments
  • Data normalization: Rust normalizes tick data into 1-minute OHLCV candles in memory, writes to TimescaleDB in batches every 5 seconds
  • Signal computation: Every new candle triggers a signal check — rule-based conditions evaluated in microseconds by Rust
  • AI enrichment: Significant signals (confidence > 70) are sent to the AI layer for plain-English annotation before displaying to user
  • Reconnection handler: If WebSocket drops, Rust attempts reconnect every 200ms, fills data gap from secondary source or flags it
  • Market close (3:30 PM): End-of-day aggregation runs, OI data reconciled against NSE official figures, daily AI debrief generated

6.3 NSE Data Sources — Free First

Important Note

NSE does not provide a free real-time API. However, several Indian brokers (Zerodha, Upstox, Dhan, Fyers) provide real-time WebSocket data free with a demat account. This is the legal, sustainable free tier data strategy.

  • Real-time: Zerodha KiteConnect (₹2,000/month) or Upstox API (free with account) — start with Upstox
  • Option chain: NSE website scrape every 3 minutes (legally permissible for personal/internal use, to be replaced with licensed feed at scale)
  • Historical EOD: NSE Bhavcopy — free daily download, automated via Python script
  • Historical intraday: TrueData (₹800/month for 1-min data, 2 years history) — Phase 2
  • India VIX: NSE website, 3-minute scrape — free
  • FII/DII data: NSE daily provisional report — free, automated download

10. Risks & Mitigations

10.1 Data Risk — NSE API Access

NSE does not provide a free real-time API. If broker API terms change, data access could be disrupted. Mitigation: Always maintain 2 data sources simultaneously. Build broker-agnostic abstraction layer in Rust so switching providers requires no frontend changes.

10.2 SEBI Regulatory Risk

Providing algorithmic trading signals to retail users may require SEBI Research Analyst registration. Mitigation: Position OPTIQ as an 'education and analytics platform' in Phase 1. Do not provide 'buy/sell' recommendations — instead provide 'signal conditions' and 'analysis'. Get RA registration before Phase 2 live signal launch.

10.3 AI Explanation Quality Risk

If the AI explanations are generic or wrong, users will churn immediately. Mitigation: Curate a library of 50 verified explanation templates for common scenarios (IV crush, gamma squeeze, trending vs. ranging market). Use Claude API with tightly structured prompts that inject real market data into every explanation.

10.4 Technical Complexity Risk

Building a Rust engine, AI layer, and real-time data pipeline simultaneously is complex for a small team. Mitigation: Phase approach. Launch with Python backtesting (acceptable for beta), Rust engine comes in Phase 2. The AI explanation feature is the priority — it requires only Python + Claude API and can be built quickly.
